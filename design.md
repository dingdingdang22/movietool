# 电影字幕智能分割工具 - 系统设计文档 (System Design)

## 1. 架构总览
本系统遵循“高内聚、低耦合”的架构原则，将整个工作流拆解为四个主要模块：**核心调度模块**、**时间轴分析模块**、**字幕重构模块**和**视频处理模块**。
各模块之间通过标准化的数据结构（如时间段对象 `Segment` 和 `SplitPlan`）进行通信，避免跨模块的直接依赖。

## 2. 核心数据结构定义
为了实现模块解耦，定义以下通用数据字典或类，作为模块间传递的“契约”：
* `TimeRange`: 包含 `start_sec` 和 `end_sec`，表示一个绝对时间段。
* `DialogueInfo`: 包含对话内容 `text` 及对应的 `TimeRange`。
* `SplitPlan`: 包含一个输出片段的元数据，如 `part_index` (第几集), `valid_time_ranges` (该集需要保留的原视频时间段列表，剔除了静音部分)。

## 3. 模块拆解与详细设计

### 3.1 时间轴分析模块 (Timeline Analyzer)
**职责**：纯粹的业务逻辑与数学计算。负责读取字幕、识别静音区间、并在弹性时长内寻找最佳切割点。
**高内聚**：只负责“算”，不涉及任何文件生成或视频剪辑。
**输入**：字幕文件路径 (`subtitle_path`)，静音阈值 (`silence_threshold`)，切片时长范围 (`min_minutes`, `max_minutes`)。
**输出**：一组 `SplitPlan` 列表，明确告知后续模块“应该怎么切”、“保留哪些时间段”。
**内部核心步骤**（支持静态方法独立解析多份字幕）：
1. **解析**：通过 `pysrt` 读取主字幕，转换为 `DialogueInfo` 列表。
2. **静音标记**：遍历相邻两句字幕，若时间差 > 静音阈值，则生成“剔除区间”。
3. **弹性切割**：累加保留区间的时长。当达到 `min_minutes` 时，开始在后续对话间隔中寻找“最长停顿”作为分割点；若达到 `max_minutes` 则强制在当前最长停顿处切割，生成一个新的 `SplitPlan`。

### 3.2 字幕重构模块 (Subtitle Reconstructor)
**职责**：根据分析模块给出的 `SplitPlan`，重新计算保留下来的字幕时间轴，并生成新的字幕文件。
**高内聚**：只关心时间戳的偏移量计算和 SRT 文本的格式化。
**低耦合**：不依赖视频文件，只要传入原字幕和 `SplitPlan` 即可独立运行。
**输入**：原字幕文件路径，`SplitPlan` 列表，输出目录。
**输出**：按集数命名的新的 `.srt` 文件（支持单字幕 `Movie_Part01.srt` 或双字幕独立输出 `_sub1.srt` / `_sub2.srt`）。
**内部核心步骤**：
1. 针对每一个 `SplitPlan`，提取主/辅字幕中其包含的原始对话。
2. 扣除被剔除静音片段的时间，将当前集的起始时间归零。
3. 将重算后的双份或单份字幕分别写出至本地。

### 3.3 视频处理模块 (Video Processor / FFmpeg Wrapper)
**职责**：执行实际的视频物理裁剪与拼接操作。
**高内聚**：封装所有 FFmpeg 的底层命令 (`subprocess` 调用)，隔离外部系统环境。
**低耦合**：不知道字幕的存在，只接收时间段指令。如果后续换成按画面黑屏切割，该模块代码一行都不用改。
**输入**：原始视频文件路径，`SplitPlan` 列表，输出目录，可选配置项（如是否压缩至 720p 适应串流）。
**输出**：按集数命名的新的 `.mp4` 文件（如 `Movie_Part01.mp4`）。
**内部核心步骤**：
1. 遍历 `SplitPlan` 中的 `valid_time_ranges`，使用 FFmpeg 的 `-ss` 和 `-to` 参数进行切割。支持两种模式：极速无损流拷贝（`-c copy`）或网络优化压缩模式（转换至 720p 并启用 `-movflags +faststart`）。
2. 如果一个 `SplitPlan` 内包含多个零碎的保留片段，使用 FFmpeg 的 `concat demuxer` 将它们无缝拼接成一集。

### 3.4 核心调度模块 (Main Controller)
**职责**：程序的入口点。负责加载配置项、初始化各模块，并按流水线顺序调度它们。
**输入**：用户指定的全局配置。支持主字幕及可选的第二字幕输入。支持 CLI 参数输入并在无参数时降级为 GUI 弹窗。
**输出**：终端的进度打印与日志监控。
**执行流程**：
1. 解析命令行参数，若为空则自动拉起 `tkinter` 交互窗口获取原始视频、字幕路径及分段阈值配置。
2. 调用 `Timeline Analyzer` 获取 `[SplitPlan_1, SplitPlan_2, ...]`。
3. 循环遍历 Plans：
   a. 异步或同步调用 `Subtitle Reconstructor` 根据输入单/双字幕情况，生成该集的一到两份字幕文件。
   b. 异步或同步调用 `Video Processor` 生成该集的视频。
4. 清理中间临时文件，输出完成报告。

## 4. 后续开发与测试建议
基于上述模块化设计，我们可以按照以下顺序进行编码与独立测试：
1. **Phase 1**: 优先开发 **时间轴分析模块**，并编写单元测试验证时间轴和分集逻辑的数学计算是否准确。
2. **Phase 2**: 开发 **字幕重构模块**，直接读取 Phase 1 的输出，生成新的字幕文件并在播放器中人工验证。
3. **Phase 3**: 开发 **视频处理模块**，实现 FFmpeg 自动化剪辑。
4. **Phase 4**: 完善 **核心调度模块**，串联完整流程。
